# 管理端上传课程视频功能实现分析

## 一、功能概述

管理端上传课程视频是在线教育平台的核心功能之一，允许管理员将教学视频上传到系统中，并与具体的课程章节进行关联。本功能采用**分片上传**技术，有效解决了大文件上传过程中的网络中断、上传失败需重新上传等问题。

## 二、系统架构

### 2.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Vue.js + Element UI |
| 后端框架 | Spring Boot + Spring Cloud |
| 数据库 | MySQL + MyBatis Plus |
| 视频处理 | FFmpeg |
| 视频流媒体 | SRS (Simple Realtime Server) |
| 消息队列 | RabbitMQ |

```
│                      mergechunks() 方法详细流程                             │
└────────────────────────────────────────────────────────────────────────────┘

  输入参数:

  - fileName: 原始文件名

  - mimetype: MIME 类型
  - fileExt: 文件扩展名
  - chapterId: 章节 ID
  - courseId: 课程 ID
  - videoNumber: 视频序号
  - name: 视频名称
  - courseName: 课程名称
  - chapterName: 章节名称

         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 1: 获取块文件文件夹路径                                      │
  │  ─────────────────────────────────────────────────────────────  │
  │  chunkfileFolderPath = /{md5[0]}/{md5[1]}/{md5}/chunks/        │
  │                                                                 │
  │  如果目录不存在，创建目录                                         │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 2: 创建合并后的文件对象                                     │
  │  ─────────────────────────────────────────────────────────────  │
  │  mergeFile = /{md5[0]}/{md5[1]}/{md5}/{md5}.{ext}              │
  │                                                                 │
  │  如果文件已存在，先删除（覆盖上传）                                │
  │  创建新文件：mergeFile.createNewFile()                         │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 3: 获取并排序所有分块文件                                   │
  │  ─────────────────────────────────────────────────────────────  │
  │  - 获取 chunks 目录下所有文件                                    │
  │  - 按文件名（分块序号）升序排序                                   │
  │  - 得到有序的 chunkFiles 列表                                     │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 4: 合并分块文件                                            │
  │  ─────────────────────────────────────────────────────────────  │
  │  使用 RandomAccessFile 按序读取每个分块，写入 mergeFile          │
  │                                                                 │
  │  伪代码：                                                        │
  │  for chunkFile in chunkFiles:                                  │
  │      while (len = raf_read.read(b)) != -1:                     │
  │          raf_write.write(b, 0, len)                            │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 5: MD5 校验文件完整性                                        │

  │  - 计算文件的 MD5 值                                              │
  │  - 与传入的 fileMd5 比较                                         │
  │                                                                 │
  │  校验失败 → 返回错误                                              │

         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐

  │  - courseId: 课程 ID                                            │
  │  - name: 视频名称                                               │
  │  - courseName: 课程名称                                         │
  │  - chapterName: 章节名称                                        │
  │  - fileUrl: 播放地址 (SRS 服务器地址 + md5.m3u8)                 │
  │  - fileStatus: 1(待处理)                                        │
  │  - number: 章节下视频序号                                       │
  └─────────────────────────────────────────────────────────────────┘
         │
```

### 3.4 视频转码推流流程

```
┌────────────────────────────────────────────────────────────────────────────┐

  │  ─────────────────────────────────────────────────────────────  │
  │  if fileType != "mp4":                                         │
  │      调用 FFmpeg 转换为 MP4 格式                                  │
  │      - 输入：原始视频文件                                        │
  │      - 输出：{md5}.mp4                                         │
  │      - 失败 → 更新 fileStatus=3(处理失败)                       │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 2: 初始化 HLS 推流工具                                      │
  │  ─────────────────────────────────────────────────────────────  │
  │  HlsVideoUtil hlsVideoUtil = new HlsVideoUtil(ffmpeg_path)    │
  │  hlsVideoUtil.init(                                            │
  │      srsRtmpPath,    // RTMP 服务器地址                          │
  │      video_path,     // MP4 文件路径                             │
  │      mediaFile.getFileId()  // 输出文件名（MD5）                 │
  │  )                                                             │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  步骤 3: 推流到 SRS 服务器                                        │
  │  ─────────────────────────────────────────────────────────────  │
  │  hlsVideoUtil.generateM3u8ToSrs()                              │
  │                                                                 │
  │  FFmpeg 命令示例：                                               │
  │  ffmpeg -i input.mp4 -c:v libx264 -c:a aac                     │
  │         -f hls -hls_time 10 -hls_list_size 0                   │
  │         rtmp://srs-server/live/{md5}                           │
  │                                                                 │
  │  生成文件：                                                      │
  │  - {md5}.m3u8  (播放列表文件)                                   │
  │  - {md5}.ts    (视频分片)                                       │
  └─────────────────────────────────────────────────────────────────┘
         │
         ▼
```

## 五、核心技术点

### 5.1 分片上传

**原理：** 将大文件按固定大小（2MB）分割成多个小块，逐块上传到服务器。

**优势：**
- 支持断点续传：网络中断后只需上传未完成的分块
- 上传进度精确：可实时显示每个分块和整体进度
- 失败重试：单个分块失败不影响其他分块
- 降低内存占用：前端无需一次性加载整个文件

**分块大小计算：**
```
总分块数 = ceil(文件大小 / 分块大小)
例如：100MB 文件，分块大小 2MB → 50 个分块
```

### 5.2 秒传（文件去重）

**原理：** 利用文件 MD5 值判断文件是否已存在。

**流程：**
1. 前端计算文件 MD5
2. 调用 `/register` 接口，传入 MD5
3. 后端检查数据库和文件系统中是否存在该 MD5 的文件
4. 如果存在，直接返回成功（秒传）
5. 如果不存在，继续正常上传流程

### 5.3 断点续传

**原理：** 记录已上传的分块，续传时跳过已完成的分块。

**流程：**
1. 文件注册后，调用 `/checkchunk` 检查每个分块
2. 构建已上传分块列表 `uploadedChunks[]`
3. 上传时跳过已存在的分块
4. 只上传未完成的部分

### 5.4 MD5 校验

**作用：** 确保合并后的文件与原始文件一致，防止传输过程中数据损坏。

**流程：**
1. 前端计算原始文件的 MD5
2. 后端合并分块后，计算合并文件的 MD5
3. 比较两个 MD5 值是否一致
4. 不一致则返回错误，需重新上传

## 九、总结

管理端上传课程视频功能采用了分片上传、秒传、断点续传等技术，有效解决了大文件上传的痛点问题。整个流程分为前端和后端两大部分：

**前端核心流程：**
1. 文件选择 → MD5 计算 → 文件注册 → 检查分块 → 分块上传 → 合并分块

**后端核心流程：**
1. 文件注册（创建目录）→ 分块保存 → 合并分块 → MD5 校验 → 数据库入库 → 异步转码推流

**技术亮点：**
- 分片上传：支持大文件稳定上传
- 秒传：相同文件无需重复上传
- 断点续传：网络中断后可继续上传
- MD5 校验：确保文件完整性
- 异步处理：转码推流不阻塞上传流程
