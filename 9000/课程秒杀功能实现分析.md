# 课程秒杀功能实现分析

## 一、功能概述

课程秒杀是在线教育平台的一种营销活动，允许用户在特定时间段内以优惠价格抢购课程。本功能采用**Redis + Redisson 信号量**技术，有效解决了高并发场景下的超卖、重复购买等问题。

### 核心特性

- **限时限量**：设置秒杀活动时间和商品库存
- **限购规则**：每人限购 1 件，防止黄牛囤货
- **库存扣减**：使用 Redisson 信号量原子操作扣减库存
- **防重复购买**：Redis 标记用户购买状态
- **预订单机制**：秒杀成功后生成预订单，30 分钟内支付

## 三、数据模型设计

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 主键 ID |

| time_str | String | 时间段显示文本（如"10:00-12:00"） |
| start_time | Date | 秒杀开始时间 |
| end_time | Date | 秒杀结束时间 |

## 六、关键技术点

### 6.1 Redisson 信号量（库存扣减）

**原理：** 使用 Redis 的原子操作实现分布式环境下的库存扣减，避免超卖问题。

**核心代码：**
```java
RSemaphore semaphore = redissonClient.getSemaphore(killCourse.getId().toString());
boolean success = semaphore.tryAcquire(1);
```

**优势：**
- 原子操作：多线程/多实例环境下保证数据一致性
- 高性能：基于 Redis，QPS 可达数万
- 无需解锁：tryAcquire 自动完成扣减

### 6.2 防重复购买

**原理：** 使用 Redis 的 Key-Value 结构记录用户购买状态。

**存储结构：**
```
Key: {userId}:{killCourseId}
Value: {orderNo}
TTL: 永久（或订单失效时间）
```

**校验逻辑：**
```java
String repeatKey = loginId + ":" + dto.getKillCourseId();
Object repeat = redisTemplate.opsForValue().get(repeatKey);
if (repeat != null) {
    throw new RuntimeException("已购买过");
}
```

### 6.3 实时状态计算

**原理：** 秒杀课程的状态（即将开始/秒杀中/已结束）根据当前时间与活动时间的比较动态计算。

**状态判断方法：**
```java
// 是否正在秒杀中
public Boolean isKilling() {
    Date now = new Date();
    return now.after(startTime) && now.before(endTime);
}

// 是否即将开始
public Boolean isUnbegin() {
    Date now = new Date();
    return now.before(startTime);
}

// 状态名称
public String getKillStatusName() {
    Date now = new Date();
    if (now.before(startTime)) return "未开始";
    if (now.after(endTime)) return "秒杀已结束";
    return "秒杀中";
}
```

### 6.4 预订单机制

**原理：** 秒杀成功后不直接创建正式订单，而是先生成预订单存储到 Redis，用户需在 30 分钟内完成支付。

**预订单数据结构：**
```java
public class PreOrderDto {
    private String orderNo;       // 订单号
    private BigDecimal totalAmount; // 订单金额
    private Integer count;        // 数量
    private Long userId;          // 用户 ID
    private Long courseId;        // 课程 ID
}
```

**存储位置：**
- Redis Key: `{orderNo}`
- Redis Value: PreOrderDto 对象 JSON

**优势：**
- 减轻数据库压力：秒杀高峰期只操作 Redis
- 超时自动取消：可设置 Redis 过期时间
- 快速响应：无需等待数据库事务

## 八、常见问题与解决方案

### 8.1 超卖问题

**问题：** 高并发下库存扣减为负数。

**解决方案：** 使用 Redisson 信号量，tryAcquire 是原子操作。

### 8.2 重复购买

**问题：** 用户同时发起多个请求，绕过限购限制。

**解决方案：** 先检查 Redis 中的购买标记，再扣减库存。

### 8.3 数据一致性

**问题：** Redis 与 MySQL 数据不一致。

**解决方案：**
- 发布活动时同步更新数据库和 Redis
- 查询时从数据库获取最新状态
- 下架活动时清除 Redis 数据

### 8.4 库存回滚

**问题：** 用户秒杀成功后未支付，库存未释放。

**解决方案：**
- 设置订单支付超时时间（30 分钟）
- 定时任务扫描超时订单，恢复库存
- 库存恢复：semaphore.release()

## 九、总结

课程秒杀功能采用了多种技术手段保证高并发场景下的系统稳定性：

**管理端核心流程：**
1. 创建秒杀活动 → 2. 添加秒杀课程 → 3. 发布活动（同步 Redis）

**用户端核心流程：**
1. 浏览列表 → 2. 查看详情 → 3. 执行秒杀 → 4. 支付订单

**技术亮点：**
- Redisson 信号量：解决超卖问题
- Redis 防重复标记：解决限购问题
- 实时状态计算：动态显示秒杀状态
- 预订单机制：减轻数据库压力
- 倒计时显示：提升用户体验
