# 在线教育平台项目技术亮点总结

## 一、项目概述

本项目是一个基于**Spring Cloud 微服务架构**的在线教育平台，采用前后端分离模式，包含课程管理、视频上传、秒杀活动、订单支付、全文搜索等核心功能模块。

### 2.2 视频分片上传与断点续传

**核心功能：** 大文件视频分片上传、MD5 校验、断点续传、秒传

**技术实现：**

```java
// 前端分片上传流程
1. 计算文件 MD5 → 2. 文件注册 → 3. 检查已传分块 → 4. 上传分块 → 5. 合并分块

// 后端合并逻辑
@Override
public JSONResult mergechunks(String fileMd5, String fileName, Long fileSize, ...) {
    // 1. 获取分块文件路径
    String chunkfileFolderPath = getChunkFileFolderPath(fileMd5);

    // 2. 获取并排序所有分块
    List<File> chunkFiles = getChunkFiles(chunkfileFolder);

    // 3. 合并分块文件
    File mergeFile = mergeFile(mergeFile, chunkFiles);

    // 4. MD5 校验文件完整性
    boolean checkResult = this.checkFileMd5(mergeFile, fileMd5);

    // 5. 保存文件信息到数据库
    mediaFileMapper.insert(mediaFile);

    // 6. 发送 MQ 消息，异步转码推流
    mediaProducer.synSend(mediaFile);
}
```

**技术亮点：**
- **分片上传**：2MB 分块，支持大文件稳定上传
- **断点续传**：检查已上传分块，只上传未完成部分
- **秒传功能**：MD5 比对，相同文件无需重复上传
- **完整性校验**：合并后 MD5 校验，确保文件完整
- **异步处理**：合并成功后通过 MQ 异步转码推流

### 2.3 秒杀系统设计与实现

**核心挑战：** 高并发、防超卖、防重复购买、库存扣减

**技术方案：**

```
┌─────────────────────────────────────────────────────────────────┐
│                      秒杀系统架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  用户请求                                                       │
│     │                                                           │
│     ▼                                                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Redis 缓存层                                            │   │
│  │  - activity:{activityId} Hash 存储商品信息               │   │
│  │  - {killCourseId} Semaphore 存储库存                    │   │
│  │  - {userId}:{killCourseId} 防重复购买标记               │   │
│  └─────────────────────────────────────────────────────────┘   │
│     │                                                           │
│     ▼                                                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Redisson 信号量扣减库存                                 │   │
│  │  boolean success = semaphore.tryAcquire(1)              │   │
│  └─────────────────────────────────────────────────────────┘   │
│     │                                                           │
│     ▼                                                           │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  生成预订单存 Redis，30 分钟内支付                        │   │
│  │  orderNo → PreOrderDto                                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**核心代码：**

```java
@Override
public String kill(KillParamDto dto) {
    // 1. 获取当前用户 ID
    Long loginId = LoginContext.getLogin().getId();

    // 2. 检查重复购买
    String repeatKey = loginId + ":" + dto.getKillCourseId();
    Object repeat = redisTemplate.opsForValue().get(repeatKey);
    AssertUtil.isNull(repeat, "已购买过");

    // 3. 查询秒杀课程
    KillCourse killCourse = onlineOne(dto.getKillCourseId(), dto.getActivityId());

    // 4. 校验秒杀时间
    AssertUtil.isTrue(killCourse.isKilling(), "不在秒杀时间");

    // 5. Redisson 信号量扣减库存
    RSemaphore semaphore = redissonClient.getSemaphore(killCourse.getId().toString());
    boolean success = semaphore.tryAcquire(1);
    AssertUtil.isTrue(success, "库存不足");

    // 6. 生成预订单
    String orderNo = CodeGenerateUtils.generateOrderSn(loginId);
    PreOrderDto preOrderDto = new PreOrderDto(...);

    // 7. 存储预订单和重复购买标记
    redisTemplate.opsForValue().set(orderNo, preOrderDto);
    redisTemplate.opsForValue().set(repeatKey, orderNo);

    return orderNo;
}
```

**技术亮点：**
- **Redis 缓存**：活动商品和库存全部存储在 Redis，减轻数据库压力
- **Redisson 信号量**：原子操作扣减库存，杜绝超卖
- **防重复购买**：Redis Key 标记用户购买状态
- **预订单机制**：秒杀成功后生成预订单，30 分钟内支付
- **实时状态计算**：动态判断秒杀状态（未开始/进行中/已结束）

### 2.4 RocketMQ 事务消息保证最终一致性

**应用场景：** 订单创建涉及多个服务（订单服务 + 支付服务）的数据一致性

**实现方案：**

```java
// 发送事务消息
@Override
public String placeOrder2(PlaceOrderDTO placeOrderDTO) {
    // 1. 准备订单数据
    CourseOrderDTO courseOrderDTO = createCourseOrders(...);
    PayOrder payOrder = createPayOrder(...);

    // 2. 发送事务消息
    TransactionSendResult result = rocketMQTemplate.sendMessageInTransaction(
        // 事务监听器组名
        RocketMQConstants.MQ_COURSEORDER_PAY_GROUP_TRANSACTION,
        // 主题：标签
        RocketMQConstants.MQ_TOPIC_ORDER + ":" + MQ_TAGS_COURSEORDER_PAYORDER,
        // 消息体（用于生成支付单）
        MessageBuilder.withPayload(JSONObject.toJSONString(payOrder)).build(),
        // 扩展参数（用于保存课程订单）
        courseOrderDTO
    );

    // 3. 检查发送结果
    if (result.getLocalTransactionState() != LocalTransactionState.COMMIT_MESSAGE) {
        throw new GlobalBusinessException("订单生成失败");
    }

    return courseOrderDTO.getCourseOrder().getOrderNo();
}
```

**事务监听器：**

```java
@Component
@RocketMQTransactionListener(txProducerGroup = "mq_courseorder_pay_group_transaction")
public class CourseOrderTransactionListener implements RocketMQLocalTransactionListener {

    // 执行本地事务
    @Override
    public RocketMQLocalTransactionState executeLocalTransaction(Message message, Object args) {
        CourseOrderDTO courseOrderDTO = (CourseOrderDTO) args;
        // 保存课程订单和订单明细
        courseOrderService.insert(courseOrderDTO.getCourseOrder());
        courseOrderItemService.insertBatch(courseOrderDTO.getCourseOrderItemList());
        return RocketMQLocalTransactionState.COMMIT;
    }

    // 事务回查
    @Override
    public RocketMQLocalTransactionState checkLocalTransaction(Message message) {
        // 查询本地订单是否存在
        CourseOrder order = courseOrderService.selectByOrderNo(payOrder.getOrderNo());
        return order != null ? COMMIT : ROLLBACK;
    }
}
```

**技术亮点：**
- **最终一致性**：通过事务消息保证订单和支付单数据最终一致
- **事务回查机制**：MQ 主动回查本地事务状态，防止消息丢失
- **解耦服务**：订单服务和支付服务通过 MQ 异步通信

### 2.5 RocketMQ 延迟消息实现订单超时取消

**应用场景：** 用户下单后 30 分钟未支付，自动取消订单

**实现方案：**

```java
// 发送延迟消息
@Override
public String placeOrder2(PlaceOrderDTO placeOrderDTO) {
    // ... 订单创建逻辑

    // 发送延迟消息（5 级延迟 = 30 分钟）
    rocketMQTemplate.syncSend(
        RocketMQConstants.MQ_TOPIC_COURSEORDER_CANCEL_DELAY + ":" +
        RocketMQConstants.MQ_TAGS_COURSEORDER_CANCEL_DELAY,
        MessageBuilder.withPayload(payOrder.getOrderNo()).build(),
        3000,  // 超时时间
        5      // 延迟等级（30 分钟）
    );

    return orderNo;
}
```

**延迟消息消费者：**

```java
@Component
```

**技术亮点：**
- **延迟消息**：利用 RocketMQ 延迟队列实现定时任务
- **自动取消**：超时订单自动取消，释放库存
- **广播模式**：确保所有消费者都能收到消息

### 2.6 基于 Elasticsearch 的课程全文搜索

**功能：** 支持多条件组合搜索、高亮显示、排序

**实现方案：**

```java
@Override
public PageList<CourseDoc> searchCourse(CourseSearchDTO param) {
    NativeSearchQueryBuilder builder = new NativeSearchQueryBuilder();

    // 1. 排序
    if (StringUtils.isNotEmpty(param.getSortField())) {
        String fieldName = getSortFieldName(param.getSortField());
        SortOrder sortOrder = "asc".equals(param.getSortType()) ? SortOrder.ASC : SortOrder.DESC;
        builder.withSort(new FieldSortBuilder(fieldName).order(sortOrder));
    }

    // 2. 分页
    builder.withPageable(PageRequest.of(param.getPage()-1, param.getRows()));

    // 3. 组合查询条件
    BoolQueryBuilder boolQueryBuilder = QueryBuilders.boolQuery();

    // 关键字匹配
    if (StringUtils.isNotEmpty(param.getKeyword())) {
        boolQueryBuilder.must(QueryBuilders.matchQuery("name", param.getKeyword()));
    }

    // 分类过滤
    if (param.getCourseTypeId() != null) {
        boolQueryBuilder.filter(QueryBuilders.termQuery("courseTypeId", param.getCourseTypeId()));
    }

    // 价格范围
    if (param.getPriceMin() != null) {
        boolQueryBuilder.filter(QueryBuilders.rangeQuery("price").gte(param.getPriceMin()));
    }

    builder.withQuery(boolQueryBuilder);

    // 4. 执行搜索（支持高亮）
    Page<CourseDoc> page = elasticsearchTemplate.queryForPage(builder.build(), CourseDoc.class);

    return new PageList(page.getTotalElements(), page.getContent());
}
```

**技术亮点：**
- **多条件组合搜索**：支持关键字、分类、价格范围等组合查询
- **高亮显示**：搜索结果关键字高亮
- **灵活排序**：支持按销量、人气、价格、上新时间排序
- **分页查询**：支持大数据量分页

### 2.7 JWT + Redis 双 Token 认证机制

**技术方案：** Access Token + Refresh Token 双令牌机制

**实现方案：**

```java
@Service
public class LoginServiceImpl implements ILoginService {

    private static final long ACCESS_TOKEN_EXPIRE_SECONDS = 7200; // 2 小时
    private static final long REFRESH_TOKEN_EXPIRE_SECONDS = 2592000; // 30 天

    @Override
    public AccessTokenVo loginCommon(LoginDTO loginDTO) {
        // 1. 验证用户名密码
        Login login = selectOne(new EntityWrapper<Login>().eq("username", loginDTO.getUsername()));

        // 2. 生成 JWT Token
        String accessToken = JwtUtil.generateAccessToken(login);
        String refreshToken = JwtUtil.generateRefreshToken(login);

        // 3. 存储到 Redis（用于主动踢人下线）
        String tokenKey = "TOKEN:" + login.getId();
        redisTemplate.opsForValue().set(tokenKey, accessToken,
            ACCESS_TOKEN_EXPIRE_SECONDS, TimeUnit.SECONDS);

        // 4. 返回 Token
        AccessTokenVo tokenVo = new AccessTokenVo();
        tokenVo.setAccessToken(accessToken);
        tokenVo.setRefreshToken(refreshToken);
        tokenVo.setExpiresTime(ACCESS_TOKEN_EXPIRE_SECONDS);

        return tokenVo;
    }

    @Override
    public AccessTokenVo refresh(RefreshTokenDTO refreshTokenDTO) {
        // 1. 验证 Refresh Token
        Login login = JwtUtil.parseToken(refreshTokenDTO.getRefreshToken());

        // 2. 生成新 Token
        String accessToken = JwtUtil.generateAccessToken(login);
        String refreshToken = JwtUtil.generateRefreshToken(login);

        // 3. 更新 Redis
        String tokenKey = "TOKEN:" + login.getId();
        redisTemplate.opsForValue().set(tokenKey, accessToken, ...);

        return tokenVo;
    }
}
```

**技术亮点：**
- **双 Token 机制**：Access Token 短期有效，Refresh Token 长期有效
- **无状态认证**：JWT 实现无状态认证，服务端不存储会话
- **主动下线**：Redis 存储 Token，支持主动删除实现踢人下线
- **自动刷新**：Access Token 过期后使用 Refresh Token 刷新

### 2.8 Spring Cache 缓存异常容错处理

**技术方案：** 自定义 CacheErrorHandler，缓存异常不影响主流程

**实现方案：**

```java
@Configuration
public class CacheConfig extends CachingConfigurerSupport {

    @Bean
    @Override
    public CacheErrorHandler errorHandler() {
        return new CacheErrorHandler() {
            @Override
            public void handleCacheGetError(RuntimeException exception, Cache cache, Object key) {
                ignoreRedisException(key, exception);
            }

            @Override
            public void handleCachePutError(RuntimeException exception, Cache cache, Object key, Object value) {
                ignoreRedisException(key, exception);
            }

            @Override
            public void handleCacheEvictError(RuntimeException exception, Cache cache, Object key) {
                ignoreRedisException(key, exception);
            }

            private void ignoreRedisException(Object key, Exception e) {
                log.error("redis operate error key:{}", key, e);
                // 缓存异常不抛出，保证主流程正常执行
            }
        };
    }
}
```

**技术亮点：**
- **异常容错**：缓存操作异常不影响主业务流程
- **优雅降级**：缓存失效时自动降级到数据库查询
- **日志记录**：记录异常便于后续排查
