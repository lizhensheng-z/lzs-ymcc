整体架构定位
本链路采用 “DB持久化 + Redis缓存预热 + 异步削峰” 的经典高并发架构，将读多写少的查询流量拦截在缓存层，核心扣减逻辑依赖 Redis 原子操作保障数据一致性，最终通过消息队列异步落库，实现抗峰值、低延迟、高可用的秒杀交易闭环。

🔹 阶段一：管理后台配置与预热
创建活动：运营通过 POST /save 提交活动基础信息（时间窗口、规则、状态），数据持久化至 t_kill_activity。
关联课程：通过 POST /save 绑定参与秒杀的课程，记录秒杀价、限购数量、库存等，持久化至 t_kill_course。
发布预热：调用 POST /publish 触发缓存预热，将活动配置、课程快照、初始库存写入 Redis activity:{id}，并初始化并发控制信号量（Semaphore），完成 DB→Cache 的数据迁移，为 C 端高并发读取做准备。
🔹 阶段二：用户端浏览与查询
列表查询：用户访问 GET /online/all，网关路由至秒杀服务，直接从 Redis 读取活动列表，避免 DB 穿透，响应时间控制在 10ms 内。
详情查看：用户点击 GET /online/one/{id}，服务从 Redis 拉取完整活动快照与课程信息，结合本地缓存策略进一步降低 Redis 压力。
🔹 阶段三：核心秒杀执行链路
执行秒杀：用户点击抢购，请求进入 POST /kill 接口，后端按序执行：
防重校验：通过 Redis SET 或 Bitmap 校验 uid 是否已参与，拦截恶意刷单与重复购买。
原子扣库存：使用 Lua 脚本或 Redisson 信号量执行 DECR 操作，确保高并发下库存不超卖、不少扣。
生成预订单：库存扣减成功后，在 Redis 中写入预订单数据（Hash/List），标记状态为 PENDING_PAY。
异步同步：将预订单投递至 MQ，消费者异步写入 MySQL 正式订单表，实现流量削峰与 DB 保护。
🔹 阶段四：支付履约与超时处理
确认支付：用户跳转支付网关完成付款。支付成功回调后，预订单转为正式订单，触发课程权限发放；若超时未支付（通常 15-30 分钟），定时任务或延迟消息触发 库存回滚 与 订单关闭，保障资金与库存一致性。



秒杀系统分层架构详细描述
🔹 1. 管理端 (ymcc-ui-system)
定位：运营配置与活动管控入口，负责秒杀活动的生命周期管理。
核心功能：
KillActivity：创建活动、设置时间窗口、上下架控制、状态流转。
KillCourse：绑定参与秒杀的课程、配置秒杀价、限购数量、初始库存。
课程管理-加入秒杀：从常规课程库中筛选商品，注入秒杀属性，完成商品池构建。
数据流向：配置数据持久化至 MySQL，点击“发布”后触发缓存预热，将活动快照与库存同步至 Redis，完成 DB → Cache 的数据迁移。
🔹 2. 用户端 (ymcc-ui-new)
定位：C 端用户交互界面，聚焦高并发场景下的流畅体验与防刷交互。
核心页面：
KillList：秒杀活动列表页，采用分页加载与本地缓存策略，降低服务端压力。
KillDetail：商品详情页，展示倒计时、库存进度、限购提示，前端内置防重复点击逻辑。
OrderConfirm：订单确认页，秒杀成功后跳转，展示预订单状态与支付倒计时。
交互特征：全链路直读 Redis，避免数据库穿透；按钮状态与库存实时联动，提升用户体验。
🔹 3. API 网关 / Feign 调用
定位：流量入口与微服务通信枢纽，承担路由、鉴权、限流、熔断职责。
核心能力：
动态路由：将 /kill/** 请求精准转发至 service-kill。
限流防刷：基于令牌桶/漏桶算法实施 IP/UID 维度限流，拦截恶意请求。
Feign 通信：秒杀服务通过 Feign 调用订单、支付、用户服务，实现业务解耦。
🔹 4. 后端服务层 (service-kill)
定位：秒杀核心业务逻辑承载层，采用 Controller + Service 分层架构，保障高可用与可维护性。
Controller 层：
KillActivityController：处理活动 CRUD、发布/下架指令，触发缓存同步。
KillCourseController：暴露列表查询、详情获取、秒杀执行接口，前置参数校验与频次控制。
Service 层：
publish()/unpublish()：管理活动状态，联动 Redis 缓存失效/重建。
onlineAll()：从 Redis Hash 批量读取活动快照，支持分页与过滤。
kill()：核心方法，执行防重校验 → 原子扣库存 → 生成预订单 → 投递 MQ 异步落库。
payKillOrder()：处理支付回调，完成订单状态流转与课程权限发放。
🔹 5. 数据存储层
MySQL：持久化活动配置、课程绑定关系、最终订单数据。采用分库分表或读写分离应对海量历史数据。
Redis：
activity:{id} Hash：缓存活动元数据与课程快照，支撑毫秒级查询。
orderNo / userId:courseId：记录预订单与用户购买状态，实现幂等与防重。
Redisson：
Semaphore {courseId}：分布式信号量替代传统分布式锁，实现高并发下的原子扣库存，避免超卖与死锁。

```java
public void publish(Long activityId) {
    // 1. 查询活动
    KillActivity killActivity = selectById(activityId);

    // 2. 查询活动下的所有秒杀课程（从DB查）
    List<KillCourse> killCourses = killCourseService.selectList(wrapper);

    // 3. 更新活动状态
    killActivity.setPublishStatus(1);  // 已发布
    updateById(killActivity);

    // 4. 遍历课程，存入Redis
    for (KillCourse killCourse : killCourses) {
        // 4.1 更新课程状态
        killCourse.setPublishStatus(1);
        killCourseService.updateById(killCourse);

        // 4.2 设置库存信号量
        RSemaphore semaphore = redissonClient.getSemaphore(killCourse.getId().toString());
        semaphore.trySetPermits(killCourse.getKillCount());

        // 4.3 存入Redis Hash
        String key = "activity:" + killActivity.getId();
        redisTemplate.opsForHash().put(key, killCourse.getId().toString(), killCourse);
    }
}
```

**Redis存储结构**：
```
Key: activity:1
Type: Hash
Value: {
  "101": KillCourse对象JSON,
  "102": KillCourse对象JSON
}

Key: 101 (课程ID)
Type: Semaphore
Value: 100 (库存数量)
```

### 步骤6：用户执行秒杀

**前端**：点击"立即秒杀"

**API**：`POST /kill/killCourse/kill`

```java
public String kill(KillParamDto dto) {
    // 1. 获取用户ID
    Long loginId = LoginContext.getLogin().getId();

    // 2. 检查是否重复购买
    String repeatKey = loginId + ":" + dto.getKillCourseId();
    Object repeat = redisTemplate.opsForValue().get(repeatKey);
    if (repeat != null) throw new RuntimeException("已购买过");

    // 3. 查询秒杀课程
    KillCourse killCourse = onlineOne(dto.getKillCourseId(), dto.getActivityId());

    // 4. 校验秒杀时间
    if (!killCourse.isKilling()) throw new RuntimeException("不在秒杀时间");

    // 5. 扣减库存（信号量）
    RSemaphore semaphore = redissonClient.getSemaphore(killCourse.getId().toString());
    boolean success = semaphore.tryAcquire(1);
    if (!success) throw new RuntimeException("库存不足");

    // 6. 生成预订单
    String orderNo = CodeGenerateUtils.generateOrderSn(loginId);
    PreOrderDto preOrder = new PreOrderDto(orderNo, killCourse.getKillPrice(), 1, loginId, killCourse.getCourseId());

    // 7. 存入Redis
    redisTemplate.opsForValue().set(orderNo, preOrder);
    redisTemplate.opsForValue().set(repeatKey, orderNo);

    return orderNo;
}
```
