## 二、数据库逻辑设计

### 2.1 逻辑设计概述

逻辑设计是将概念模型转换为特定DBMS所支持的数据模型的过程。本系统采用关系型数据库MySQL，将E-R模型转换为关系模式。

### 2.2 关系模式转换规则

1. **实体转换**：每个实体转换为一个关系（表）
2. **属性转换**：实体的属性转换为关系的属性（字段）
3. **主键转换**：实体的标识符转换为关系的主键
4. **关系转换**：
   - 一对一关系：在任一方添加外键
   - 一对多关系：在"多"方添加外键
   - 多对多关系：创建关联表

## 三、数据库物理设计

### 3.1 数据库选型

本系统采用 **MySQL 5.7** 作为数据库管理系统，主要原因如下：

1. **开源免费**：降低项目成本
2. **性能优异**：支持高并发读写
3. **生态完善**：与Spring Cloud微服务架构集成良好
4. **社区活跃**：问题容易得到解决

### 3.2 数据库分布策略

采用**分库分表**策略，按业务模块划分数据库：

| 数据库名 | 模块 | 说明 |
|---------|------|------|
| ymcc-user | 用户模块 | 用户相关数据 |
| ymcc-uaa | 认证授权模块 | 登录、权限相关数据 |
| ymcc-system | 系统管理模块 | 系统配置、员工、部门 |
| ymcc-course | 课程模块 | 课程、分类、章节等 |
| ymcc-media | 媒体模块 | 视频文件信息 |
| ymcc-order | 订单模块 | 订单数据 |
| ymcc-pay | 支付模块 | 支付数据 |
| ymcc-kill | 秒杀模块 | 秒杀活动数据 |
| ymcc-cart | 购物车模块 | 购物车数据 |

### 3.3 存储引擎选择

| 存储引擎 | 适用场景 | 本系统应用 |
|---------|---------|-----------|
| InnoDB | 需要事务支持、外键约束 | 所有业务表 |
| MyISAM | 只读或读多写少 | 日志表、统计表 |

**选择InnoDB的原因：**
1. 支持事务（ACID），保证数据一致性
2. 支持外键约束，保证数据完整性
3. 支持行级锁，提高并发性能
4. 支持崩溃恢复

### 3.4 字符集与排序规则

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 字符集 | utf8 | 支持中文存储 |
| 排序规则 | utf8_general_ci | 不区分大小写排序 |

### 3.5 数据类型选择原则

| 数据类型 | 适用场景 | 示例 |
|---------|---------|------|
| BIGINT | 主键、外键、大数值 | id, user_id |
| INT | 数量、计数 | count, sale_count |
| TINYINT | 状态、类型、开关 | status, type |
| DECIMAL | 金额、价格 | price, amount |
| VARCHAR | 可变长字符串 | name, phone |
| TEXT/LONGTEXT | 大文本内容 | description, intro |
| DATETIME | 日期时间 | create_time |
| DATE | 日期 | birthday |
| BIT | 布尔值 | enabled, free |

## 五、数据库设计总结

### 5.1 设计特点

1. **模块化设计**：按业务领域划分数据库，实现微服务架构下的数据隔离
2. **规范化设计**：遵循数据库三范式，减少数据冗余
3. **适度反范式**：在订单明细等表中保留冗余字段，提高查询性能
4. **统一命名规范**：表名、字段名采用统一的命名规则，提高可读性
5. **完善的索引设计**：主键、外键、唯一约束、普通索引合理配置

### 5.2 数据库规模统计

| 模块 | 数据库 | 表数量 | 说明 |
|------|--------|--------|------|
| 用户模块 | ymcc-user | 6 | 用户、账户、地址、实名、流水 |
| 认证授权模块 | ymcc-uaa | 7 | 登录、角色、权限、菜单、OAuth |
| 系统管理模块 | ymcc-system | 5 | 员工、部门、配置、字典 |
| 课程模块 | ymcc-course | 8 | 课程、分类、章节、详情、营销等 |
| 媒体模块 | ymcc-media | 1 | 媒体文件 |
| 订单模块 | ymcc-order | 2 | 订单、订单明细 |
| 支付模块 | ymcc-pay | 3 | 支付订单、流水、支付宝配置 |
| 秒杀模块 | ymcc-kill | 2 | 秒杀活动、秒杀课程 |
| 购物车模块 | ymcc-cart | 1 | 购物车 |
| **合计** | **9个数据库** | **35张表** | - |

### 5.3 关键技术点

1. **共享主键**：用户与账户、课程与详情等一对一关系采用共享主键设计
2. **自关联设计**：部门、菜单、课程分类采用自关联实现树形结构
3. **中间表设计**：用户-角色、角色-权限等多对多关系通过中间表实现
4. **乐观锁设计**：订单表使用version字段实现乐观锁，防止并发问题
5. **时间戳设计**：部分表使用BIGINT存储时间戳，便于跨时区处理
