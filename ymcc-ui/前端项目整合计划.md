## 二、整合方案

### 2.1 推荐方案：整合为2个项目

```
整合后:
├── ymcc-ui (用户端 + 课程前端)  - 端口 6002
│   ├── views/
│   │   ├── course/      (原 course 页面)
│   │   ├── user/       (原 user 页面)
│   │   └── common/     (公共页面)
│   └── public/         (静态资源)
│
└── ymcc-ui-admin (管理后台)     - 端口 8081
    └── (保持现有 system 不变，或合并)
```

**说明**:
- 用户端和课程前端整合为一个Vue项目 (ymcc-ui)
- 管理后台保持独立 (ymcc-ui-admin)

## 六、风险和注意事项

### 6.1 主要风险

1. **样式兼容性** - 原 HTML 项目使用 jQuery + 原生 JS，部分样式可能需要调整
2. **API 差异** - 三个项目使用相同的 API 基础 URL，但部分接口可能不同
3. **登录状态** - 需要统一 Token 存储方式

### 6.2 注意事项

1. **API 统一配置**
   - 统一使用 `http://127.0.0.1:10010/ymcc` 作为基础 URL
   - 统一 Token 存储到 localStorage

2. **组件复用**
   - 课程卡片组件在列表页和用户首页复用
   - 订单卡片组件在订单列表和详情页复用

3. **渐进式迁移**
   - 不要求一次性完成所有页面迁移
   - 可以先完成核心页面，其他页面逐步迁移

## 七、总结

整合后的优势:
- 统一技术栈，便于维护和扩展
- 减少项目数量，降低部署复杂度
- 代码复用，提高开发效率
- 统一登录状态，用户体验更好

建议先执行整合计划，后续再考虑是否将管理后台也整合进来。
