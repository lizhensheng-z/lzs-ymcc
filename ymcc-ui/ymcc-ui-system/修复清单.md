# ymcc-ui-system 前端页面修复清单
 注意：原则上尽量不要改动后端代码！！！如果有需要，
 
可以新增后端接口
## 项目架构说明

### 后端网关路由映射
| 前端路径前缀 | 后端服务 | 说明 |
|------------|---------|------|
| `/ymcc/user/**` | service-user | 用户服务 |
| `/ymcc/uaa/**` | service-uaa | 认证授权服务 |
| `/ymcc/course/**` | service-course | 课程服务 |
| `/ymcc/system/**` | service-system | 系统服务 |
| `/ymcc/order/**` | service-order | 订单服务 |
| `/ymcc/pay/**` | service-pay | 支付服务 |
| `/ymcc/kill/**` | service-kill | 秒杀服务 |
| `/ymcc/common/**` | service-common | 公共服务 |

### 后端接口规范
所有 Controller 遵循统一的 RESTful 接口规范：
- `POST /xxx/save` - 保存/修改数据
- `DELETE /xxx/{id}` - 删除数据
- `GET /xxx/{id}` - 获取单条数据
- `GET /xxx/list` - 获取列表数据
- `POST /xxx/pagelist` - 分页查询数据

### 分页参数规范
- `page` - 当前页码（从1开始）
- `rows` - 每页条数（默认10）
- `keyword` - 查询关键字

### 1. KillActivity.vue - 语法错误
**文件路径**: `src/views/itsource/KillActivity.vue`

**问题描述**: 第184行有多余字符 `f`

**代码位置**:
```javascript

add: function () {

    this.addForm.sex = -1;
    this.addForm.age = 0;

```

**修复方案**:
```javascript
add: function () {
    this.addFormVisible = true;
    for(let p in this.addForm){
        this.addForm[p] = '';
    }
    this.addForm.sex = -1;
    this.addForm.age = 0;
},
```

### 3. KillActivityController.java - 删除接口方法不匹配
**文件路径**: `ymcc-service/ymcc-service-kill/src/main/java/cn/lzs/ymcc/web/controller/KillActivityController.java`

**问题描述**: 删除接口没有指定 HTTP 方法，但前端使用 POST 调用

**后端代码** (第47行):
```java
@RequestMapping(value="delete/{id}")
public JSONResult delete(@PathVariable("id") Long id){
```

**前端调用** (KillActivity.vue 第217行):
```javascript
this.$http.post("/kill/killActivity/delete/" + id)
```

**修复方案** (二选一):

**方案A** - 修改后端，支持 POST:
```java
@PostMapping("delete/{id}")
public JSONResult delete(@PathVariable("id") Long id){
```

**方案B** - 修改前端，使用 GET:
```javascript
this.$http.get("/kill/killActivity/delete/" + id)
```

**推荐方案A**，保持与其他删除接口一致（使用 POST）

### 5. CourseType.vue - 响应数据层级错误
**文件路径**: `src/views/itsource/CourseType.vue`

**问题描述**: 第156行访问 `res.data.data.data` 层级错误，应该是 `res.data.data`

**代码位置**:
```javascript
// 第154-157行
getChildrenByPid(pid){
    this.$http.get("/course/courseType/selectChildrenById/"+pid).then(res=>{
        this.datas = res.data.data.data;  // <-- 层级错误
    });

```

**修复方案**:
```javascript
getChildrenByPid(pid){
    this.$http.get("/course/courseType/selectChildrenById/"+pid).then(res=>{
        this.datas = res.data.data;
    });
},
```

**注意**: 需要确认后端 `CourseTypeController` 是否有 `selectChildrenById` 接口，如果没有需要添加。

- `src/views/itsource/Systemdictionary.vue` (第128-131行)
- `src/views/itsource/Systemdictionaryitem.vue` (第122-125行)
- `src/views/itsource/role.vue` (第109-112行)
- `src/views/itsource/Permission.vue` (第134-137行)
- `src/views/itsource/Teacher.vue` (第144-147行)
- `src/views/itsource/PayFlow.vue` (第152-155行)
- `src/views/itsource/Alipay.vue` (第139-142行)
- `src/views/itsource/SMSMessage.vue` (第136-139行)
- `src/views/itsource/EmailMessage.vue` (第137-140行)
- `src/views/itsource/StationMessage.vue` (第140-143行)
- `src/views/itsource/KillActivity.vue` (第160-163行)
- `src/views/itsource/CourseKill.vue` (第124-127行)

**问题描述**: 分页查询时只传递了 `page` 和 `keyword`，缺少 `rows` 参数

**示例代码** (Employee.vue):
```javascript
let para = {

    keyword: this.filters.keyword
    // 缺少 rows 参数
};
```

```javascript
let para = {
    page: this.page,
    rows: 20,  // 添加每页条数，与 el-pagination 的 :page-size 一致
    keyword: this.filters.keyword
};
```

| StationMessage.vue | 20 | 20 |

| CourseKill.vue | 10 | 10 |
| CouseOrder.vue | 10 | 10 |
