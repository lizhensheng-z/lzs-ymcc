│   ├─────────────────────────────────────────────────────────────────────────────────┤   │   │
│   │                                                                                 │   │
│   │   检查时机: 每次发送请求前                                                  │   │
│   │                                                                                 │   │                                                                               
│   │   ┌─────────────────────────────────────────────────────────────────────────┐   │   │                                                                                 
│   │   │  刷新逻辑:                                                               │   │   │
│   │   │                                                                         │   │   │                                                                                 
│   │   │  1. 获取本地存储的过期时间                                   │   │   │
│   │   │  2. 计算当前时间与过期时间的差值                                           │   │   │
│   │   │  3. 如果差值 < 5分钟，自动刷新Token                                        │   │   │
│   │   │                                                                         │   │   │

│   │   │  diff = (expiresTime - nowTime) / 1000 / 60  (分钟)                      │   │   │
│   │   │                                                                         │   │   │
│   │   │  判断:                                                                   │   │   │
│   │   │  if (diff <= 0 || diff < 5) {                                            │   │   │
│   │   │      // Token已过期或即将过期，刷新Token                                  │   │   │
│   │   │      doRequest();                                                        │   │   │
│   │   │  }                                                                       │   │   │

七、完整认证流程总结

┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                              完整认证流程总结                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────────────────┐
│                                                                                          │
│   ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│   │                              登录阶段                                             │   │
│   ├─────────────────────────────────────────────────────────────────────────────────┤   │   │
│   │                                                                                 │   │
│   │   1. 用户输入凭证 (用户名/密码)                                                   │   │
│   │   2. 前端发送登录请求到网关                                                       │   │
│   │   3. 网关白名单放行，转发到UAA服务                                                │   │
│   │   4. UAA服务验证凭证:                                                            │   │
│   │      - 查询用户信息                                                              │   │
│   │      - 验证用户类型                                                              │   │
│   │      - 验证密码                                                                  │   │
│   │      - 检查账户状态                                                              │   │
│   │   5. 生成JWT Token (access_token + refresh_token)                                │   │
│   │   6. 存储Token到Redis                                                            │   │                                                                              
│   │   7. 返回Token给前端                                                             │   │                                                                                
│   │   8. 前端存储Token到localStorage                                                 │   │
│   │   9. 跳转到对应页面                                                              │   │                                                                                
│   │                                                                                 │   │
│   └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                          │
│   ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│   │                              请求阶段                                             │   │
│   ├─────────────────────────────────────────────────────────────────────────────────┤   │   │
│   │                                                                                 │   │
│   │   1. 前端检查Token是否即将过期                                                    │   │
│   │   2. 如果即将过期，自动刷新Token                                                  │   │
│   │   3. 请求携带Authorization头                                                     │   │
│   │   4. 网关拦截请求:                                                               │   │
│   │      - 检查是否在白名单                                                          │   │
│   │      - 提取Token                                                                 │   │
│   │      - 验证Token签名和有效期                                                      │   │
│   │      - 解析用户信息                                                              │   │
│   │      - 将用户信息添加到请求头                                                     │   │                                                                             
│   │   5. 转发请求到下游服务                                                          │   │                                                                                
│   │   6. 下游服务从请求头获取用户信息                                                 │   │
│   │   7. 处理业务逻辑                                                                │   │                                                                                
│   │   8. 返回响应                                                                    │   │
│   │                                                                                 │   │
│   └─────────────────────────────────────────────────────────────────────────────────┘   │
│                                                                                          │
│   ┌─────────────────────────────────────────────────────────────────────────────────┐   │
│   │                              安全特性                                             │   │
│   ├─────────────────────────────────────────────────────────────────────────────────┤   │   │
│   │                                                                                 │   │
│   │   ✓ 密码加密存储 (BCrypt)                                                        │   │
│   │   ✓ Token签名防篡改 (HMAC-SHA256)                                               │   │
│   │   ✓ Token过期机制 (Access: 2小时, Refresh: 30天)                                 │   │
│   │   ✓ 网关统一验证，下游服务无感知                                                  │   │
│   │   ✓ 用户类型隔离 (用户端/管理端)                                                 │   │
│   │   ✓ 白名单机制 (公开接口无需认证)                                                │   │
│   │   ✓ 自动Token刷新 (无感续期)                                                     │   │
