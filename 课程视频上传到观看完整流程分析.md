```javascript
// 计算文件 MD5（用于文件去重和分块管理）
async calculateMd5(file) {
  const chunkSize = 2 * 1024 * 1024; // 2MB
  const chunks = Math.ceil(file.size / chunkSize);
  const spark = new SimpleMD5.ArrayBuffer();

  // 逐块计算 MD5
  for (let i = 0; i < chunks; i++) {
    const start = i * chunkSize;
    const end = Math.min(start + chunkSize, file.size);
    const chunk = file.slice(start, end);
    // ... 读取并累加 MD5
  }
  return spark.end(); // 返回最终 MD5 值
}
```

#### 步骤 2: 文件注册

**接口**: `POST /media/mediaFile/register`

**参数**:
- `fileMd5`: 文件 MD5 值
- `fileName`: 文件名
- `fileSize`: 文件大小
- `mimetype`: MIME 类型
- `fileExt`: 文件扩展名

**后端处理**: `MediaFileServiceImpl.register()`

```java
@Override
public JSONResult register(String fileMd5, String fileName, Long fileSize, String mimetype, String fileExt) {
    // 检查文件是否已存在
    String filePath = getFilePath(fileMd5, fileExt);
    File file = new File(filePath);

    // 查询数据库
    MediaFile media = mediaFileMapper.selectById(fileMd5);

    // 文件和记录都存在则返回已存在
    if(file.exists() && media != null){
        return JSONResult.error("上传文件已存在");
    }

    // 创建文件目录
    createFileFold(fileMd5);

    return JSONResult.success();
}
```

#### 步骤 3: 检查已上传分块（断点续传）

**接口**: `POST /media/mediaFile/checkchunk`

**参数**:
- `fileMd5`: 文件 MD5
- `chunk`: 分块索引
- `chunkSize`: 分块大小

**后端处理**: `MediaFileServiceImpl.checkchunk()`

```java
@Override
public JSONResult checkchunk(String fileMd5, Integer chunk, Integer chunkSize) {
    String chunkfileFolderPath = getChunkFileFolderPath(fileMd5);
    File chunkFile = new File(chunkfileFolderPath + chunk);

    if(!chunkFile.exists()){
        return JSONResult.error(); // 分块不存在
    }
    return JSONResult.success(); // 分块已存在，可跳过
}
```

#### 步骤 4: 分块上传

**接口**: `POST /media/mediaFile/uploadchunk`

**参数**:
- `file`: 分块文件
- `fileMd5`: 文件 MD5
- `chunk`: 分块索引

**后端处理**: `MediaFileServiceImpl.uploadchunk()`

```java
@Override
public JSONResult uploadchunk(MultipartFile file, String fileMd5, Integer chunk) {
    // 创建分块文件目录
    createChunkFileFolder(fileMd5);

    // 保存分块文件（以索引命名，无扩展名）
    File chunkfile = new File(getChunkFileFolderPath(fileMd5) + chunk);

    InputStream inputStream = file.getInputStream();
    FileOutputStream outputStream = new FileOutputStream(chunkfile);
    IOUtils.copy(inputStream, outputStream);

    return JSONResult.success();
}
```

#### 步骤 5: 合并分块

**接口**: `POST /media/mediaFile/mergechunks`

**参数**:
- `fileMd5`: 文件 MD5
- `fileName`: 文件名
- `fileSize`: 文件大小
- `mimetype`: MIME 类型
- `fileExt`: 文件扩展名
- `courseId`: 课程 ID
- `chapterId`: 章节 ID
- `videoNumber`: 视频序号
- `name`: 视频名称
- `courseName`: 课程名称
- `chapterName`: 章节名称

**后端处理**: `MediaFileServiceImpl.mergechunks()`

```java
@Override
public JSONResult mergechunks(String fileMd5, String fileName, Long fileSize,
                              String mimetype, String fileExt,
                              Long courseChapterId, Long courseId,
                              Integer number, String name,
                              String courseName, String chapterName) {

    // 1. 获取分块文件路径
    String chunkfileFolderPath = getChunkFileFolderPath(fileMd5);
    File chunkfileFolder = new File(chunkfileFolderPath);

    // 2. 创建合并后的文件
    File mergeFile = new File(getFilePath(fileMd5, fileExt));

    // 3. 合并所有分块
    List<File> chunkFiles = getChunkFiles(chunkfileFolder); // 已排序
    mergeFile = mergeFile(mergeFile, chunkFiles);

    // 4. 校验 MD5
    boolean checkResult = checkFileMd5(mergeFile, fileMd5);
    if(!checkResult){
        return JSONResult.error("文件校验失败！");
    }

    // 5. 保存媒体文件记录到数据库
    MediaFile mediaFile = new MediaFile();
    mediaFile.setFileId(fileMd5);
    mediaFile.setFileName(fileMd5 + "." + fileExt);
    mediaFile.setFileOriginalName(fileName);
    mediaFile.setFilePath(getFileFolderRelativePath(fileMd5, fileExt));
    mediaFile.setFileSize(fileSize);
    mediaFile.setMimeType(mimetype);
    mediaFile.setFileType(fileExt);
    mediaFile.setChapterId(courseChapterId);
    mediaFile.setCourseId(courseId);
    mediaFile.setName(name);
    mediaFile.setCourseName(courseName);
    mediaFile.setChapterName(chapterName);
    mediaFile.setFileUrl(srsPalyPath + mediaFile.getFileId() + ".m3u8");

    // ✅ 修复后：状态设置为 2（处理成功）
    mediaFile.setFileStatus(2);

    // 6. 生成视频序号
    Wrapper<MediaFile> wrapper = new EntityWrapper<>();
    wrapper.eq("course_id", courseId);
    wrapper.eq("chapter_id", courseChapterId);
    Integer videoCount = mediaFileMapper.selectCount(wrapper);
    mediaFile.setNumber(videoCount + 1);

    // 7. 插入数据库
    mediaFileMapper.insert(mediaFile);

    // 8. MQ 推送断点续播消息
    boolean success = mediaProducer.synSend(mediaFile);

    return success ? JSONResult.success() : JSONResult.error();
}
```

**数据库表**: `t_media_file`

| 字段 | 说明 | 示例值 |

| file_id | 文件 MD5 | abc123... |
| file_name | 存储文件名 | abc123....mp4 |

```java
@Override
public List<CourseChapter> getCourseChaptersByCourseId(Long courseId) {
    // 1. 查询所有章节
    List<CourseChapter> courseChapters = listByCourseId(courseId);
    if (courseChapters == null) {
        return Collections.emptyList();
    }

    // 2. 远程调用获取媒体文件（只返回 fileStatus=2 的文件）
    List<MediaFile> mediaFiles = null;
    try {
        JSONResult mediaFilesByCourseId = mediaFileApi.getMediaFilesByCourseId(courseId);
        Object data = mediaFilesByCourseId.getData();
        String jsonString = JSONObject.toJSONString(data);
        mediaFiles = JSONObject.parseArray(jsonString, MediaFile.class);
    } catch (Exception e) {
        // 异常处理
    }

    // 3. 将章节与视频配对
    courseChapters.forEach(chapter -> {
        List<MediaFile> files = Optional.ofNullable(mediaFiles)
                .orElse(Collections.emptyList())
                .stream()
                .filter(Objects::nonNull)
                .filter(file -> Objects.equals(file.getChapterId(), chapter.getId()))
                .collect(Collectors.toList());
        chapter.setMediaFiles(files);
    });

    return courseChapters;
}
```

**媒体服务查询**: `MediaFileServiceImpl.getMediaFilesByCourseId()`

```java
@Override
public List<MediaFile> getMediaFilesByCourseId(Long courseId) {
    if(courseId == null){
        throw new GlobalBusinessException("参数异常");
    }
    // ✅ 修复后：只查询处理成功的文件 (fileStatus=2)
    List<MediaFile> mediaFiles = mediaFileMapper.selectList(
        new EntityWrapper<MediaFile>()
            .eq("course_id", courseId)
            .eq("file_status", 2)  // 关键：只返回状态为 2 的文件
    );
    return mediaFiles;
}
```

#### 步骤 3: HLS 视频播放

**前端播放器**: 使用 `hls.js` 库

```javascript
import Hls from 'hls.js'

initVideoPlayer(videoUrl) {
  const video = this.$refs.videoPlayer;
  if (!video) return;

  // 判断是否是 HLS (m3u8) 格式
  if (videoUrl.indexOf('.m3u8') > -1) {
    if (Hls.isSupported()) {
      // 浏览器支持 HLS
      const hls = new Hls();
      hls.loadSource(videoUrl); // 加载 m3u8 索引文件
      hls.attachMedia(video);   // 绑定到 video 元素
      hls.on(Hls.Events.MANIFEST_PARSED, () => {
        video.play();
      });
      this.hls = hls;
    } else if (video.canPlayType('application/vnd.apple.mpegurl')) {
      // Safari 原生支持 HLS
      video.src = videoUrl;
    }
  } else {
    // 普通视频直接设置 src
    video.src = videoUrl;
  }
}
```

## 五、已修复的问题

### 5.1 问题 1: 媒体文件状态设置错误

**问题描述**: 管理端上传成功后，用户端查询章节时 `mediaFiles` 为空数组

**根本原因**: `mergechunks` 方法保存媒体文件时设置 `fileStatus=1`（处理中），而查询时只返回 `fileStatus=2`（处理成功）的文件

**修复位置**: `MediaFileServiceImpl.java` 第 238-239 行

**修复前**:
```java
mediaFile.setFileStatus(1); // ❌ 错误：状态为处理中
```

**修复后**:
```java
mediaFile.setFileStatus(2); // ✅ 正确：状态为处理成功
```

### 5.2 问题 2: 查询未过滤文件状态

**问题描述**: `getMediaFilesByCourseId` 方法查询所有状态的文件，可能返回处理中或失败的文件

**修复位置**: `MediaFileServiceImpl.java` 第 90-95 行

**修复前**:
```java
List<MediaFile> mediaFiles = mediaFileMapper.selectList(
    new EntityWrapper<MediaFile>().eq("course_id", courseId)
);
```

**修复后**:
```java
List<MediaFile> mediaFiles = mediaFileMapper.selectList(
    new EntityWrapper<MediaFile>()
        .eq("course_id", courseId)
        .eq("file_status", 2)  // ✅ 只返回处理成功的文件
```

## 九、可能的问题排查点

### 问题 1: 上传成功但用户端看不到视频

**排查步骤**:
1. 检查数据库中 `file_status` 是否为 2
2. 检查 `chapter_id` 是否正确关联
3. 检查 Feign 远程调用是否成功
4. 查看日志是否有异常

### 问题 2: 视频无法播放

**排查步骤**:
1. 检查 `file_url` 是否为有效的 m3u8 地址
2. 检查 SRS 流媒体服务器是否正常运行
3. 检查转码是否完成（查看 `handleFile2m3u8` 执行日志）
4. 检查浏览器是否支持 HLS（IE 不支持）

### 问题 3: 上传失败

**排查步骤**:
1. 检查磁盘空间是否充足
2. 检查上传目录是否有写权限
3. 检查 ffmpeg 是否安装并配置正确路径
4. 查看 `mergechunks` 方法日志
