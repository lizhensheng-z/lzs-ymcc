# 在线教育平台单元测试用例说明

### 1.1 测试目的

本项目编写的单元测试旨在验证各核心服务模块的功能正确性，确保代码质量，提高系统的可维护性和稳定性。

| 课程服务 | CourseServiceTest | ymcc-service-course/src/test/java/cn/lzs/ymcc/service/ |

- **测试框架**: JUnit 4

## 六、测试注意事项

### 6.1 数据库隔离

- 所有测试使用 `@Transactional` 注解，测试结束后自动回滚
- 测试前使用 `@Before` 清理数据，测试后使用 `@After` 清理资源

### 6.2 Redis 数据

- 测试中使用的 Redis Key 应使用唯一前缀
- 测试结束后应清理 Redis 数据，避免影响其他测试

### 6.3 测试数据

- 使用 `TestDataUtils` 生成随机数据，避免数据冲突
- 测试数据应有明确的含义，便于问题排查

### 6.4 断言编写

- 断言消息应清晰描述预期结果
- 使用有意义的断言，便于定位问题
